## 背景與目標

在日外國人（俗稱 gaijin）面對龐雜且快速變動的制度環境：簽證規則每年修訂、銀行政策因國籍不同而異、稅務申告流程對非日語使用者極不友善。現有資訊散落在 Reddit 串文、Facebook 社群、政府 PDF、KOL 部落格，缺乏結構化整合。

**目標**：建立一個「活的」知識庫，讓 AI 助手能準確、及時地回答在日外國人的實際問題，並隨環境變化自動更新。

**核心假設**：
- 在日外國人最高頻問題集中在 8 大類別
- 60% 的問題有明確的「正確答案」（法規/流程），40% 需要情境判斷
- 知識有效期差異極大（簽證規則年度更新 vs 日常生活技巧長期有效）

### 持續更新機制

#### 自動監控觸發條件
```
IF 官方來源出現關鍵詞變化 (新規定/改正/令和X年度)
  → 自動標記相關知識條目為 "PENDING_REVIEW"
  → 發送 Slack/Telegram 通知給編輯團隊

IF 社群討論量突增（某主題 24h 內新增 >20 則討論）
  → 自動生成「熱點話題」摘要草稿
  → 人工審核決定是否轉為正式知識條目
```

#### 季節性更新日曆
| 時期 | 重點更新領域 |
|------|-------------|
| 1-2 月 | 確定申告・年末調整相關 |
| 3-4 月 | 年度制度更新（新財年） |
| 6-7 月 | 住民稅通知・夏季更新 |
| 10-11 月 | 社會保険・年金年度確認 |
| 隨時 | 簽證制度變更（無規律） |

## 品質評分機制

### 三維評分模型

#### 維度 1：時效性評分（Timeliness Score, T）

```
T = base_score × decay_factor

base_score:
  - 官方來源: 1.0
  - 知名媒體: 0.9
  - 社群精選: 0.7
  - UGC: 0.5

decay_factor（依知識類型）:
  - VOLATILE（法規/簽證）: T = base × e^(-λt), λ=0.1/月
    → 6個月後降至 0.55, 12個月後降至 0.30
  - ANNUAL（稅務/年金）: 每年4月重置, 前一年資料降至 0.3
  - EVERGREEN（文化/基礎常識）: 幾乎不衰減, λ=0.01/月
```

#### 維度 2：可信度評分（Reliability Score, R）

| 來源類型 | 基礎分 | 加分條件 |
|----------|--------|---------|
| 政府官方網站 | 1.00 | — |
| 律師/稅務師撰寫 | 0.95 | +0.05 若有資格認證 |
| 知名媒體 | 0.85 | +0.05 若有編輯審核標記 |
| 高讚社群討論 | 0.70 | +0.10 若有專業人士確認 |
| 個人部落格 | 0.60 | +0.15 若多方交叉驗證 |
| 匿名社群回覆 | 0.40 | +0.10 若與官方資訊吻合 |

**交叉驗證加分**：同一事實有 3+ 獨立來源確認 → +0.10

#### 維度 3：適用性評分（Applicability Score, A）

根據使用者 Profile 計算：
```
A = audience_match × region_match × nationality_match

audience_match: 1.0（完全吻合）/ 0.7（部分相關）/ 0.3（不太相關）
region_match: 1.0（同地區）/ 0.8（全國適用）/ 0.5（其他地區）
nationality_match: 1.0（適用）/ 0.5（需確認）/ 0.0（不適用）
```

#### 綜合評分（Composite Score）

```
Q = 0.35×T + 0.40×R + 0.25×A
```

**評分閾值**：
- Q ≥ 0.80 → ✅ 可直接使用
- 0.60 ≤ Q < 0.80 → ⚠️ 使用但附加免責聲明
- Q < 0.60 → ❌ 禁止使用，標記為待審核

### 自動降級觸發

- 任何官方來源同主題出現更新 → 相關條目自動降為 PENDING
- 社群出現大量「這個資訊過時了」的回覆 → 觸發人工複核
- 條目超過 `next_review` 日期仍未複核 → 自動降評分

## 競品分析

### GaijinPot

**優點**：
- 覆蓋面廣（求職、租屋、語言學習一站式）
- 中英日三語（部分）
- 社群活躍（論壇有歷史積累）

**知識庫差距**：
- ❌ 知識嚴重過時（部分文章仍是 2018 年資訊）
- ❌ 無版本控制或最後更新標記
- ❌ 無個人化（所有人看到相同內容）
- ❌ 搜尋功能差，難以找到特定情境答案
- ❌ 無法回答「如果我的情況是 X，那我該怎麼做？」
- ❌ 廣告導向，內容商業化嚴重

**我們的機會**：時效性保證 + AI 情境化回答

---

### Savvy Tokyo

**優點**：
- 女性視角深度內容
- 文章品質高、有作者署名
- 生活風格內容有溫度

**知識庫差距**：
- ❌ 受眾過窄（主要針對英語系女性）
- ❌ 缺乏法規/行政類實用內容
- ❌ 無結構化 FAQ，純文章形式
- ❌ 更新頻率不穩定
- ❌ 無繁體中文/韓文支援

**我們的機會**：多語言 + 全受眾覆蓋

---

### JapanVisitor / Japan-Guide

**優點**：
- 旅遊相關資訊完整
- SEO 排名高

**知識庫差距**：
- ❌ 定位是遊客，不是長期居民
- ❌ 完全缺乏簽證/稅務/銀行等居民核心需求
- ❌ 無社群互動
- ❌ 內容深度不足

**我們的機會**：居民導向（非遊客）的深度知識

## 技術架構建議

### 知識庫存儲
```
PostgreSQL（主存儲）
  ├── knowledge_entries（主表）
  ├── knowledge_sources（來源關聯）
  ├── knowledge_versions（版本歷史）
  └── quality_scores（評分記錄）

Qdrant / Weaviate（向量存儲）
  └── 知識條目 Embedding（用於語義搜尋）

Redis（快取）
  └── 高頻問題回答快取（TTL: 24h）
```

### 採集 Pipeline
```
Scrapy / Playwright（爬蟲）
  → Apache Kafka（訊息佇列）
  → Python 處理器（清洗/評分）
  → 人工審核介面（Airtable/Notion）
  → PostgreSQL + Qdrant
```

### AI 回答系統
```
使用者輸入
  → Profile 解析（國籍/簽證/地區）
  → 向量搜尋（Top 5 相關條目）
  → 品質過濾（Q < 0.60 排除）
  → LLM 生成回答（GPT-4o / Claude）
  → 來源引用附加
  → 回答輸出 + 免責聲明（若 Q < 0.80）
```

## KPI 與成功指標

### 知識庫健康指標
- **覆蓋率**：8 大類別各有 ≥ 30 條有效條目
- **時效率**：90 天內未更新的 VOLATILE 條目比例 < 10%
- **品質率**：Q ≥ 0.80 的條目佔比 > 70%

### AI 助手表現指標
- **準確率**：人工抽樣評估，目標 ≥ 85%
- **召回率**：用戶問題能找到相關知識的比例 > 90%
- **用戶滿意度**：5 星評分系統，目標平均 ≥ 4.2

### 運營指標
- **知識更新延遲**：官方規定更新後 72 小時內完成知識庫更新
- **社群熱點響應**：熱點出現後 48 小時內有對應知識條目

## 風險與挑戰

### 風險 1：法規資訊快速變動
**影響**：高  
**對策**：自動監控 + 醒目的「最後驗證日期」標示，建立快速更新 SOP

### 風險 2：知識條目因國籍不同而差異大
**影響**：高  
**對策**：nationality_specific 標籤 + 分流回答邏輯，優先覆蓋最大群體（中/韓/英語系）

### 風險 3：用戶依賴 AI 做法律/財稅決策
**影響**：極高  
**對策**：強制加入免責聲明，複雜案件推薦諮詢專業人士（附行政書士/稅務師找法）

### 風險 4：社群知識品質參差不齊
**影響**：中  
**對策**：嚴格的 Tier 2/3/4 採集審核，不允許未審核的 UGC 直接進入知識庫

### 風險 5：資源不足以維持知識更新
**影響**：中  
**對策**：MVP 期優先自動化 Tier 1 採集，Tier 2/3 採用社群貢獻者機制
